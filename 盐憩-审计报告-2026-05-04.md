**日期**: 2026-05-04  
**范围**: `site/` 下全部 7 个 HTML、9 个 CSS、32 个 JS 文件  
**方法**: 三线并行只读审计（HTML 结构/语义、CSS 架构/特异性、JS 逻辑/选择器交叉验证）  
**排除**: home.html 第 80-101 行弯引号问题（已修复）

## P0 — 严重（阻塞渲染或大面积失效）

### 1. page-transition.css L112：content 值缺少闭合引号，导致文件尾部全部失效

**文件**: `site/css/page-transition.css`，第 112 行

```css
.page-transition-spotlight::before {
    content: ";
```

`content: "` 是一个未闭合的字符串字面量。经全文件检索，该文件共出现三处双引号：第 57 行 `content: ""`、第 70 行 `content: ""`、第 112 行 `content: ";` —— 第 112 行的开引号之后再无闭合引号，直到文件末尾。

**影响**：CSS 解析器在遇到 `content: "` 之后，将第 112 行至文件末尾的全部内容视为 `content` 属性值的字符串，所有位于其后的 CSS 规则（过渡动画 keyframes、page-stage 布局、navbar 过渡状态、海流效果等）均不会被解析为有效规则。Chrome 可能在 `}` 或 EOF 处自动恢复，但 Safari 和 Firefox 的严格解析器会丢弃文件尾部。跨页面过渡动画（页面切换时"下潜/上浮"的核心体验）在严格浏览器中完全失效。

**修复**: `content: ";` → `content: "";`

## P1 — 高（功能受损或可靠性缺口）

### 2. contact.html：表单字段缺少 `required` 属性

**文件**: `site/contact.html`

| 字段 | 行号 | 缺失 |
|------|------|------|
| `#contactName` | 132 | `required` + `pattern` |
| `#contactTopic` | 140 | `required`（select 可空提交） |
| `#contactMessage` | 149 | `required` + `maxlength` |

表单已设 `novalidate` 交由 JS 校验，但缺失 HTML 级约束意味着 JS 失败时无兜底。`#contactMessage` 作为主留言字段缺少 `maxlength`，不受限输入可能超出 localStorage 容量。

### 3. index.html L291：注册协议复选框缺少 `required`

**文件**: `site/index.html`，第 291 行

```html
<input type="checkbox" id="agree-terms">
```

注册表单的"我已阅读并同意用户协议与隐私政策"复选框未标记 `required`。auth.js 虽在提交前校验，但缺失原生约束意味着绕过 JS 校验时可跳过协议同意直接提交。

### 4. index.html L234：`aria-describedby` 在注册模式下指向错误的标题

**文件**: `site/index.html`，第 234 行

```html
<form ... aria-describedby="authModeSummaryLogin" novalidate>
```

表单的 `aria-describedby` 始终指向 `authModeSummaryLogin`（"欢迎回来"），即使用户已切换至注册标签页。处于注册模式时，屏幕阅读器用户会听到"欢迎回来"而非"留下你的手机号"。JS 应在标签切换时动态更新 `aria-describedby` 指向。

## P2 — 中（布局/行为异常或兼容性缺陷）

### 5. `.hero-content` 跨文件类名冲突

**文件**: `site/css/detail.css` L458 vs `site/css/home.css` L889

- `detail.css` L458：`.hero-content` 定义为绝对定位浮层（`position: absolute; bottom: 0; left: 0; right: 0`），嵌在 `.detail-hero` 内部
- `home.css` L889：`.hero-content` 定义为 `display: grid` 的内容容器，含 `max-width: 1240px`

`trip.html` 同时加载 `home.css` 和 `trip.css`，`detail.html` 也加载 `home.css`。若 detail 页使用 `.hero-content` 类名，会继承 home.css 的 `display: grid` 规则，导致绝对定位布局异常。建议为 detail.css 的 `.hero-content` 增加父级限定（如 `.detail-hero .hero-content`）。

### 6. 全局缺少 favicon 和 meta description

**影响范围**: 全部 7 个 HTML 文件

- 无 `<link rel="icon">` 标签，浏览器每次请求 `/favicon.ico` 返回 404
- 无 `<meta name="description">` 标签，影响 SEO 预览和社交分享卡片的文案提取

### 7. trip.html L95-97：`search-bar` div 的嵌套结构问题

**文件**: `site/trip.html`

```html
<!-- L95 --> <div class="planner-capsule trip-reveal-block" id="plannerCapsule">
<!-- L96 -->     <div class="search-bar trip-search-bar planner-desk-control" id="plannerDeskControl">
<!-- L97 -->     <div class="planner-console-head">
```

第 97 行 `planner-console-head` 的缩进与第 96 行 `search-bar` 相同，浏览器将其解析为 `search-bar` 的兄弟节点（而非子节点）。`search-bar` div 实际上包装了空内容（L281 闭合前仅含空白），真正的控制台内容（console-head、tide-flow、三个规划字段、提交按钮）均直接挂在 `planner-capsule` 下。若存在 `.search-bar .planner-console-head` 等后代选择器，将无法匹配。

### 8. home.css L2869：缺少 `-webkit-backdrop-filter`

**文件**: `site/css/home.css`，第 2869 行

```css
backdrop-filter: blur(8px);
```

该处缺少对应的 `-webkit-backdrop-filter: blur(8px);`。项目中其他所有 `backdrop-filter` 声明均已配套 `-webkit-` 前缀，此处为孤立遗漏，会在 Safari < 15 中导致毛玻璃效果失效。

## P3 — 低（代码质量、维护性或潜在风险）

### 9. JS：多处匿名事件监听器无法清理

| 文件 | 行号 | 事件 | 说明 |
|------|------|------|------|
| auth.js | 1897 | window.resize | 匿名箭头函数，不可 remove |
| depth-manager.js | 3357, 1417 | window.resize | 两处匿名箭头函数 |
| detail.js | 15554-15563 | window.resize | 三处监听，其中一处匿名 |
| detail.js | 15564 | window.scroll | 每次滚动调用 `closeBookingMatchConfirmation`，无节流 |
| trip.js | 3176, 3217 | yanqi:confirmed-bookings-updated | 两处匿名回调 |
| home.js | 3694 | window.resize | TodaySeaBrief 类内匿名函数 |
| info-pages.js | 418 | window.scroll | 每次滚动执行 `querySelectorAll` |

在当前的完整页面加载架构下这些问题不造成实际泄漏，但若未来转向 SPA 模式会逐步积累。

### 10. CSS：home.css 中存在大量死代码

- `.hero-section` 第 389-400 行（被第 729-740 行完全覆盖）
- `.hero-content` 第 432-438 行（被第 889-896 行完全覆盖）
- `body.home-page .hero-section` 在文件中被重复定义约 11 次，每次仅修改少量属性，产生大量冗余

### 11. global.css：未使用的 CSS 自定义属性

**文件**: `site/css/global.css`，第 33-34 行

```css
--glass-bg: rgba(255, 255, 255, 0.7);
--glass-backdrop: blur(20px);
```

全项目搜索，这两个变量未被任何 CSS 规则引用。所有毛玻璃效果使用 `--surface-backdrop-*` 变量族，此处为早期版本遗留。

### 12. detail.js L2990：`this.spotData.depth` 缺少空值防护

```js
syncDepthGaugeProfile() {
    if (!window.DepthManager ...) { return; }
    window.DepthManager.setDetailGaugeProfile(this.spotData.depth);
}
```

`this.spotData` 未做空值检查。若异步数据加载失败，访问 `.depth` 将抛出 `TypeError`。

### 13. home.js / detail.js：模板字面量选择器缺少 CSS.escape()

**文件**: `site/js/home.js` L16271，`site/js/detail.js` 同类位置

```js
registerPointerWarmup(`.related-feature-card[data-id="${spot.id}"]`);
```

`spot.id` 直接拼入 CSS 选择器字符串。当前 ID 均为数字，安全；若未来数据来源扩展至用户输入或 API，可能产生选择器注入。

### 14. terms.html / privacy.html：注释中残留 `??` 占位符

**文件**: `site/terms.html` L5，`site/privacy.html` L5

```html
<!-- 阅读顺序建议：?? -->
```

两处 HTML 注释的阅读建议内容均以 `??` 结尾，应为未完成编辑的占位符。
