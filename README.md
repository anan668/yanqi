# 盐憩

盐憩是一个潜水旅游主题网页项目。

它不是普通旅游信息站，也不是工具型平台，而是一个围绕“海层体验、品牌气质和沉浸式浏览节奏”设计出来的前端项目。这个网站希望让用户在浏览、选择和规划之间，像在同一片海里缓慢移动，而不是在几张普通网页之间来回跳转。

## 项目定位

- 品牌名：盐憩
- 项目类型：潜水旅游主题网页
- 核心体验：从海面入口开始，被海吸引，慢慢靠近，再继续下潜到更适合自己的那一片海
- 关键词：深海、安静、高级、舒缓、沉浸、呼吸感、玻璃感、雾感

页面层级关系不是普通并列导航，而是海层关系：

- `index.html`：潜前门厅，进入盐憩前的一层静水
- `home.html`：浅层入口，先看海、先靠近海
- `trip.html`：更深一层的规划空间，把海收进行程
- `detail.html`：进入一片具体海域，理解它的节奏和性格
- `contact.html` / `terms.html` / `privacy.html`：更浅、更静的信息说明层

## 核心系统

### 深度计系统

深度计不是装饰，而是整站的空间语言。

它负责表达：

- 页面层级的深浅关系
- 滚动时 section 之间的海层变化
- 跨页时的下潜 / 上浮感
- 某些模块切换时的轻微深度浮动

### 页面切换系统

页面切换不是普通 fade，而是更接近“潜浮式”过渡：

- `home -> trip`：继续下潜
- `trip -> home`：缓慢上浮
- `detail -> detail`：像在相邻海域之间潜游和平移

### 本地存储系统

当前项目是纯前端站点，没有接数据库和后端订单系统，所以部分状态保存在浏览器本地。

主要 key：

- `YANQI_PLANNER_DRAFT`：保存 trip 页当前的海域、日期、人数草稿
- `YANQI_CONFIRMED_BOOKINGS`：保存 detail 页确认收进行程的套餐
- `YANQI_CONTACT_MESSAGES`：保存 contact 页前端演示版留言

注意：这些数据只保存在当前浏览器中，清缓存、换浏览器或换设备后不会同步。

## 页面说明

### `index.html`

登录 / 注册页。它不是普通 auth 模板，而是“进入盐憩前的一层静水门厅”。

### `home.html`

首页。负责建立盐憩世界观，让用户先被海吸引，再慢慢进入具体目的地和匹配逻辑。

### `trip.html`

行程页。负责把灵感转化为安排，是“更深一层”的规划空间。

### `detail.html`

详情页。负责让用户真正进入某一片海域，查看潜点、套餐、评论、地图和相关推荐。

### `contact.html`

联系我们页面。角色是“静水联络台”，不是普通客服页。

### `terms.html`

用户协议页面。角色是“进入盐憩之前的使用约定说明页”。

### `privacy.html`

隐私政策页面。角色是“关于信息如何被安静保管的一页说明”。

## 运行方式

### 直接打开

这是一个静态前端项目，基础浏览可以直接双击 HTML 文件打开。

### 本地服务器

如果要避免某些浏览器对本地脚本、资源或跨页行为的限制，建议起一个本地静态服务器：

```powershell
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000/index.html
```

## 建议测试路径

如果要快速理解项目当前状态，建议按下面顺序测试：

1. 打开 `index.html`
   - 看登录页是否仍然是“潜前门厅”而不是普通认证页
2. 进入 `home.html`
   - 看 Hero、今日海域舞台、Curated Waters 和 Dive Match
3. 进入某个 `detail.html?id=...`
   - 看英雄区、预订区、评论区、地图区、海图导览、相关推荐和详情页 footer
4. 在 detail 页确认一个套餐
   - 看是否出现“已收进行程”的柔和反馈
5. 再去 `trip.html`
   - 看“已经收进行程的海”是否正确读取刚确认的套餐
6. 查看 `contact.html` / `terms.html` / `privacy.html`
   - 确认这些入口已在登录页和全站 footer 接通

## 当前实现重点

目前项目已经明确的方向包括：

- 首页不是普通旅游首页，而是“浅海入口”
- 首页 `Dive Match` 不是筛选器，而是“更适合你的那一片海”
- Trip 页不是普通表单，而是“更深一层的规划海域”
- Detail 页不是普通景点详情，而是“进入一片海”
- 详情页确认套餐后，不自动强制跳走，而是先给出“已收进行程”的安静反馈
- “已经收进行程的海”不是订单列表，而是海域档案卡
- 信息页不是普通白底法务页，而是盐憩世界里的“信息层”

## 维护原则

后续继续修改这个项目时，优先遵守这些原则：

1. 先保证品牌感和海洋叙事感
2. 再补功能，而不是直接退回普通网页方案
3. 避免把模块做成：
   - 普通搜索栏
   - 普通推荐区
   - 普通地图占位
   - 普通下拉菜单
   - 普通页脚
   - 普通后台组件
4. 所有重要改动都优先考虑是否和：
   - 深度计
   - 页面切换
   - 海图导览
   - 页面层级关系
   保持一致

如果一个模块看起来像普通旅游网站、电商站或后台系统，通常意味着它还没有真正回到“盐憩”的世界观里。
